Fase 1 — Infraestructura

eureka-server

Service Discovery — registro y localización dinámica de todos los microservicios

:8761 sin BD
Lo que hace
  • Cada servicio se registra aquí al arrancar
  • El Gateway lo consulta para resolver nombres a IPs
  • Dashboard en http://localhost:8761
Configuración clave
  • @EnableEurekaServer en la clase principal
  • eureka.client.register-with-eureka=false
  • eureka.client.fetch-registry=false

api-gateway

Punto de entrada único — enruta, valida JWT y aplica filtros

:8080 sin BD
Lo que hace
  • Recibe todas las peticiones externas
  • Redirige a cada servicio según el path
  • Valida token JWT antes de reenviar
Rutas principales
/api/auth/**→ auth-service
/api/clientes/**→ user-service
/api/vehiculos/**→ ms-vehiculos
/api/espacios/**→ ms-espacios
/api/reservas/**→ ms-reservas
/api/accesos/**→ ms-accesos
/api/tarifas/**→ ms-tarifas
/api/cobros/**→ ms-pagos
/api/reportes/**→ ms-reportes
Fase 2 — Autenticación (implementar primero)

auth-service

Autenticación JWT — desbloquea la seguridad de todos los demás servicios

:8081 db_auth
Entidades
Rol UserCredential
Endpoints
POST/api/auth/login→ devuelve JWT
POST/api/auth/register→ crea credencial
Clases clave
  • JwtUtil — genera y valida tokens (JJWT 0.11.5)
  • JwtAuthFilter — OncePerRequestFilter
  • SecurityConfig — rutas públicas / protegidas
  • BCrypt strength 12 para passwords
⚠ Agregar JJWT manualmente al pom.xml — no está en Initializr

Fórmula de cobro
monto_base = precio_base_hora
× multiplicador_horario
× factor_tipo_vehiculo
× factor_tipo_espacio
× (minutos / 60)

monto_final = monto_base
× (1 - desc_tipo_cliente / 100)
× (1 - desc_suscripcion / 100)
× (1 - desc_banco / 100)
