🏗️ Requisitos de Arquitectura (Mínimo 10 Microservicios)

Para que el proyecto sea válido, el ecosistema debe contar con al menos 10 microservicios funcionando de forma coordinada. Dentro de estos 10, existen 3 servicios obligatorios de infraestructura de seguridad:

    🔐 auth-service (Autenticación): Responsable del Login, generación de JWT y validación de credenciales. (Endpoints: /auth/login, /auth/validate).
    👤 user-service (Usuarios): Responsable del registro de nuevos usuarios y gestión de sus datos personales.
    🛡️ security-service (Roles y Permisos): Gestión de roles (ADMIN, CLIENTE, etc.) y lógica de autorización/permisos.

A estos 3 se deben sumar al menos 7 microservicios de lógica de negocio (ej: pedidos, stock, facturación, notificaciones, etc.).

Si un grupo elige un sistema de gimnasio, el flujo de pensamiento debería ser el siguiente:

 3. El Flujo de Comunicación (La lógica de clase)

    El Cliente (Torniquete): El socio pone su huella en la entrada. El hardware hace una petición al API-Gateway (8080).
    API-Gateway: Redirige la petición al ms-control.
    ms-control (El Orquestador):

    Llama vía Feign al ms-socios para saber quién es el dueño de esa huella.
    Llama vía Feign al ms-cuotas para preguntar: "¿Este socio tiene la mensualidad pagada?".

    Respuesta: Si ambos responden OK, el ms-control envía la orden de abrir el torniquete.


🛠️ Estructura del Proyecto para los Alumnos

Para que el proyecto sea válido según lo aprendido, deben entregar:

    1 Eureka Server: (Puerto 8761) Donde todos se ven.
    1 API Gateway: (Puerto 8080) Por donde entra la información.
    10 o más Microservicios de Negocio: Cada uno con su propia base de datos en XAMPP (Puerto 3307).
    Comunicación Feign: Una carpeta client en el microservicio que necesite consultar datos de otro.

Ejemplos de Temas Sugeridos:

    Clínica: ms-pacientes y ms-reservas-medicas.
    Biblioteca: ms-libros y ms-prestamos.
    Veterinaria: ms-mascotas y ms-fichas-clinicas.
    E-commerce de Ropa: ms-catalogo y ms-carrito.


⚠️ Recordatorio Técnico para los Grupos:

Recuerden configurar el puerto de MySQL en sus application.properties como localhost:3307 y asegurarse de que el API Gateway use Netty (Spring Cloud Gateway) y no Tomcat (WebMVC), para evitar el error 404 que corregimos anteriormente.
