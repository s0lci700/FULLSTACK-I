Estimados/as estudiantes:

Con motivo del cierre de la Evaluación Parcial 3 (Lunes 29/06 a las 23:59 hrs), es imperativo establecer un estándar profesional en el uso de los repositorios de Git. El almacenamiento indiscriminado de archivos binarios o instaladores en GitHub degrada el rendimiento del control de versiones y constituye una mala práctica arquitectónica.

Por lo tanto, la entrega se regirá bajo los siguientes lineamientos estrictos de separación de entornos:

1. 🛑 REGLA DE ORO: ¿QUÉ VA EN GIT Y QUÉ SE EXCLUYE?

SÍ DEBE ESTAR EN GIT (Código Fuente): La estructura Maven Multi-Módulo completa (proyecto-parent/ y carpetas hijas), archivos pom.xml, archivos fuente .java, properties/yml y configuraciones globales (incluyendo el archivo docker-compose.yml base).
❌ NO DEBE SUBIRSE A GIT (Instaladores y Binarios): Está estrictamente prohibido subir archivos .jar, carpetas target/, instaladores empaquetados, ejecutables .bat / .sh o carpetas de datos de bases de datos persistentes.
Recomendación: Verifiquen que su archivo .gitignore excluya las carpetas target/ antes del congelamiento del repositorio.

📄 2. REQUISITO OBLIGATORIO: SECCIÓN SUPERIOR DEL README.MD

Para que el cliente final (y el docente) pueda evaluar la entrega de forma autónoma y con un "solo clic", el archivo README.md localizado en la raíz del repositorio de GitHub deberá contar de forma obligatoria, en su sección superior (al principio de todo), con accesos directos e independientes almacenados externamente en Google Drive (u otra nube pública con accesos abiertos).

Deberán maquetar los enlaces utilizando la siguiente estructura limpia:

Markdown

# 🚀 SISTEMA DE MICROSERVICIOS MULTIMÓDULO - ENTREGA FINAL

🛠️ 3. CHECKLIST OPERATIVO DE LOS ENTREGABLES EN DRIVE

Asegúrense de que los archivos .zip subidos a su nube contengan exactamente la estructura limpia conversada en los manuales de despliegue:

ZIP Sin Docker (Nativo): Debe contener una carpeta /apps con los componentes requeridos (eureka-server.jar, api-gateway.jar, ms-xxx.jar) y el archivo de automatización por tiempos arrancar-nativo.bat. Requiere Java y MySQL configurados en el sistema anfitrión.
ZIP Con Docker (Avance): Debe contener la carpeta /apps con los .jar, las instrucciones de orquestación en el docker-compose.yml (con volumen persistente para MySQL) y el script definitivo arrancar-sistema.bat. Solo requiere Docker Desktop en la máquina del cliente.
