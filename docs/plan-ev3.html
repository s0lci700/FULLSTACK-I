Preparación defensa escrita (01/07 en adelante)

Las dos deben poder explicar estos temas sin ayuda. El docente genera preguntas personalizadas basadas en el repo.

Fórmula de cobro (ms-pagos)

monto_base = precio_hora × mult_horario × factor_vehiculo × factor_espacio × (min/60)
monto_final = monto_base × (1 - desc_cliente) × (1 - desc_suscripcion) × (1 - desc_banco)
Todo en BigDecimal con HALF_UP.

Estado machine de reservas

PENDIENTE → CONFIRMADA o CANCELADA
CONFIRMADA → FINALIZADA (solo desde accesos/salida)
No hay marcha atrás desde CANCELADA ni FINALIZADA.

Por qué nunca @PatchMapping

El HttpURLConnection que usa Feign por defecto no soporta el método HTTP PATCH. Además, el docente solo enseñó PUT. Por eso todas las actualizaciones parciales usan @PutMapping.

JWT — flujo completo

Solo auth-service emite tokens (login → JwtUtil.generateToken).
El API Gateway los valida en JwtAuthFilter antes de rutear.
Rutas públicas: /api/auth/**, /swagger-ui/**.

Base de datos por servicio

10 bases MySQL separadas, sin FK entre bases. Las referencias cruzadas se guardan como Long idXRef con @Column — no @ManyToOne ni @JoinColumn cross-service.
