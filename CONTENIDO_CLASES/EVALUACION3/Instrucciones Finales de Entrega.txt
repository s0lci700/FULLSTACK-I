🛑 1. REGLA DE ORO: BUENAS PRÁCTICAS EN GITHUB (¿QUÉ VA Y QUÉ SE EXCLUYE?)

El almacenamiento indiscriminado de archivos binarios o instaladores en GitHub degrada el rendimiento del control de versiones y constituye una mala práctica arquitectónica.

SÍ DEBE ESTAR EN GIT (Código Fuente): La estructura Maven Multi-Módulo completa (proyecto-parent/ y carpetas hijas), archivos pom.xml, código fuente .java, properties/yml, pruebas unitarias y configuraciones globales (incluyendo el archivo docker-compose.yml base).
❌ NO DEBE SUBIRSE A GIT (Instaladores y Binarios): Está estrictamente prohibido subir archivos .jar, carpetas target/, instaladores empaquetados, ejecutables .bat / .sh o carpetas con datos locales de bases de datos.
Recomendación: Verifiquen que su archivo .gitignore excluya correctamente las carpetas target/ antes del congelamiento del repositorio.
