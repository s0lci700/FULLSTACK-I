Guía de pruebas manuales

El problema más frecuente al testear manualmente es que el token JWT obtenido en el login no se propaga automáticamente al resto de las requests. Esta sección explica dos métodos para resolverlo.

¿Por qué pasa esto?

Postman no captura ni guarda el token automáticamente. Si solo copias el token una vez y lo pegas en cada request a mano, cualquier nueva ejecución del login genera un token diferente y todas las requests con el token viejo devuelven 401 Unauthorized.

Configura el request de login para que capture el token automáticamente y lo guarde como variable de colección. Con esto el token siempre estará actualizado sin intervención manual.

Paso a paso

  1. Abrir el request de Login (request #1) En tu colección Postman, selecciona el request POST /api/auth/login.
  2. Ir a la pestaña "Scripts" → "Post-response" En versiones anteriores de Postman esta pestaña se llama "Tests". Es el script que se ejecuta después de recibir la respuesta.
  3. Pegar el siguiente script
    Post-response Script 
    const json = pm.response.json();
if (json.token) {
    pm.collectionVariables.set("token", json.token);
    console.log("Token guardado:", json.token.slice(0, 40) + "...");
}
    Esto guarda el campo token de la respuesta como variable de la colección.
  4. Declarar la variable en la colección Clic derecho en la colección → Edit → pestaña Variables. Agrega una fila: nombre token, valor inicial vacío. Guarda.
  5. Agregar el header de autorización a cada request protegido Existen dos formas — elige la que prefieras:
    Opción rápida — Header directo en cada request
    Key:   Authorization
Value: Bearer {{token}}
    Opción elegante — Authorization a nivel de colección (se hereda en todos los requests)
    Colección → Edit → Authorization
  Type:  Bearer Token
  Token: {{token}}
  6. Ejecutar el login y luego el resto de requests Envía primero el request #1. Una vez que responda 200, {{token}} tiene el JWT y todos los requests que lo referencian quedan autorizados automáticamente.
Verificar que funcionó

Abre la consola de Postman (Ctrl+Alt+C) y busca la línea "Token guardado: …". También puedes ir a la colección → Variables y ver el Current Value de token.

Método más simple pero más tedioso: copias el token de la respuesta y lo pegas a mano en una variable de entorno. Funciona bien para pruebas puntuales donde no quieres scripts.

  1. Crear un entorno de Postman Clic en el ícono de entornos (engranaje ⚙ arriba a la derecha) → Add. Nómbralo Estacionamiento Local.
  2. Agregar las variables del entorno
    Variables a crear
    base   →  http://localhost:8080
token  →  (dejar vacío por ahora)
  3. Seleccionar el entorno activo En el dropdown superior derecho de Postman, selecciona Estacionamiento Local.
  4. Enviar el login y copiar el token Ejecuta POST /api/auth/login. En la respuesta verás un JSON con campo token. Copia el valor completo (es una cadena larga que empieza con eyJ...).
    eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJhZG1pbkBwYXJraW5nLmNsIi... ← ejemplo
  5. Pegar el token en la variable de entorno Ve a entornos → Estacionamiento Local → columna Current Value de la fila token → pegar → guardar.
  6. Agregar el header Authorization en los requests
    Header que va en cada request protegido
    Authorization: Bearer {{token}}
    O configúralo a nivel de colección (Edit → Authorization → Bearer Token → {{token}}) para que se herede en todos.
Importante

El token expira en 24 horas (jwt.expiration=86400000). Si ves 401 inesperados al día siguiente, repite el paso 4 y 5.

Swagger UI tiene su propio mecanismo de autorización. Útil para pruebas rápidas directamente en el navegador sin Postman.

  1. Obtener el token desde auth-service Abre http://localhost:8081/swagger-ui/index.html → endpoint POST /api/auth/login → Try it out → ejecutar con {"email":"admin@parking.cl","password":"Test1234!"}. Copia el valor del campo token de la respuesta.
  2. Abrir la página Swagger del servicio que quieres probar Accede directamente al puerto del servicio, no por el gateway. Por ejemplo para ms-reservas: http://localhost:8086/swagger-ui/index.html.
  3. Clic en "Authorize" (candado 🔒) Está arriba a la derecha de la página Swagger. Se abre un diálogo de autorización.
  4. Ingresar el token en el campo Bearer
    Formato en el campo "Value"
    Bearer eyJhbGciOiJIUzI1NiJ9.eyJzdWIi...
    Prefija con Bearer (con espacio). Clic en Authorize → Close.
  5. Ejecutar los endpoints Todos los endpoints de esa página Swagger ahora usarán el token. El token sólo aplica a la pestaña actual del navegador — si abres otro servicio en Swagger debes repetir el paso 4.
