1. Descripción de la tarea

El objetivo de esta unidad es integrar Swagger / OpenAPI 3 en cada microservicio para que los endpoints queden documentados de forma automática e interactiva.

  • 1
    Dependencia ya presente
    springdoc-openapi-starter-webmvc-ui:2.8.6 en los 10 servicios funcionales (eureka y gateway no necesitan Swagger).
  • 2
    Crear SwaggerConfig.java por servicio
    Un @Bean OpenAPI que define el título, versión y descripción del API de cada microservicio. Se crea en el paquete config de cada servicio.
  • 3
    Anotar controladores con @Tag
    Se agrega @Tag(name, description) a nivel de clase en cada @RestController para agrupar los endpoints en la UI.
  • 4
    Anotar métodos con @Operation y @ApiResponse
    Cada método HTTP recibe @Operation(summary, description) y @ApiResponse(responseCode, description) para describir la operación y sus posibles respuestas.

2. Patrón aplicado

SwaggerConfig.java (por servicio)

@Configuration
public class SwaggerConfig {

    @Bean
    public OpenAPI customOpenAPI() {
        return new OpenAPI().info(new Info()
            .title("API Estacionamiento - Nombre Servicio")
            .version("1.0.0")
            .description("Descripción del servicio"));
    }
}

Controlador anotado

@Tag(name = "Vehículos", description = "Gestión de vehículos del sistema")
@RestController
@RequestMapping("/api/vehiculos")
public class VehiculoController {

    @Operation(summary = "Listar vehículos", description = "Retorna todos los vehículos registrados")
    @ApiResponse(responseCode = "200", description = "Operación exitosa")
    @GetMapping
    public ResponseEntity<List<VehiculoResponseDTO>> listar() { ... }

    @Operation(summary = "Crear vehículo", description = "Registra un nuevo vehículo")
    @ApiResponse(responseCode = "201", description = "Vehículo creado correctamente")
    @ApiResponse(responseCode = "409", description = "Patente ya registrada")
    @PostMapping
    public ResponseEntity<VehiculoResponseDTO> crear(...) { ... }
}
