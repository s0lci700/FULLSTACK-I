## Design Context

### Users
Primary users are student team members developing and testing the microservices project.  
They use the HTML interfaces as operational documentation to run services, verify APIs, and navigate architecture/testing assets quickly during implementation and debugging.

### Brand Personality
Technical, clear, trustworthy.  
The interface should communicate confidence and precision, with low ambiguity and strong information hierarchy.

### Aesthetic Direction
Dark-first UI, aligned with GitHub Docs/GitHub dark visual language.  
Use compact, documentation-oriented layouts (cards/tables/labels), restrained accent colors for state and emphasis, and monospace treatment for endpoints/commands/IDs.

### Design Principles
1. Prioritize scanability over decoration: headings, chips, and tables should make operational actions obvious at a glance.
2. Keep visual semantics consistent: color/state mappings (success/warning/error/info) must remain stable across all docs pages.
3. Optimize for technical workflows: API paths, ports, commands, and references should be prominent and copy-friendly.
4. Preserve dark-theme readability: maintain strong text contrast and avoid low-contrast secondary content in critical sections.
5. Prefer predictable structure: similar page types should share layout patterns so users can transfer familiarity across documents.
